## App Router란?

Next.js 13에서 도입된 App Router는 `app/` 디렉토리 기반의 새로운 라우팅 시스템입니다.

## 서버 컴포넌트

App Router에서 모든 컴포넌트는 **기본적으로 서버 컴포넌트**입니다.

```tsx
// 이 컴포넌트는 서버에서 실행됩니다
export default async function Page() {
  const data = await fetch("https://api.example.com/posts");
  const posts = await data.json();

  return (
    <ul>
      {posts.map((post) => (
        <li key={post.id}>{post.title}</li>
      ))}
    </ul>
  );
}
```

클라이언트 컴포넌트가 필요하면 `"use client"` 지시어를 사용합니다.

```tsx
"use client";

import { useState } from "react";

export function Counter() {
  const [count, setCount] = useState(0);
  return <button onClick={() => setCount(count + 1)}>Count: {count}</button>;
}
```

## 레이아웃

`layout.tsx`는 하위 라우트에서 공유되는 UI를 정의합니다.

```tsx
export default function BlogLayout({ children }: { children: React.ReactNode }) {
  return (
    <div className="max-w-2xl mx-auto">
      <nav>Blog Navigation</nav>
      {children}
    </div>
  );
}
```

## 정리

- **서버 컴포넌트**: 데이터 페칭, 보안, 성능 최적화
- **클라이언트 컴포넌트**: 인터랙션, 브라우저 API
- **레이아웃**: 공유 UI, 상태 유지
